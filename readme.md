# Проект приюта-приемника для собак

Данный проект включает в себя 3 основных раздела:
1) Работа с питомцами
2) Личный кабинет пользователя
3) Работа с отзывами

Виртуальное окружение используемое для проекта: .venv

1) После настройки виртуального окружения установите зависимости из файла requirements.txt
 ```bash 
    pip install -r requirements.txt
 ```
2) Заполните .env согласно .env_sample
3) Создайте базу данных при помощи команды:
```bash
    python manage.py ccdb
```
4) Создайте миграции при помощи: 
```bash
    python manage.py makemigrations
```
5) Проведите миграции при помощи: 
```bash
    python manage.py migrate
```
6) Выполните команду для создания пользователей с разным уровнем доступа при помощи: 
```bash
    python manage.py ccsu
```
7) Выполните команду для заполнения БД изпользуя фикстуры при помощи: 
```bash
    python manage.py loaddata dogs_fixture.json 
    python manage.py loaddata reviews_fixture.json  
```
8) Выполните команду для Redis при помощи:
```bash
    redis-server
```
9) Откройте второе окно терминала и выполните команду:
```bash
    python manage.py runserver
```

Модели используемые в проекте:
1) Dogs с полями:
- name - Кличка;
- breed - Порода;
- photo - Фото питомца;
- birth_date - дата рождения питомца;
- is_active - Активность;
- owner - Хозяин;
- views - Просмотры;

2) Reviews с полями:
- title - Заголовок; 
- slug - URL;
- content - Содержимое; 
- created - Создан;
- sign_of_review - Активность;
- author - Автор;
- dog - Собака;

3) Users с полями:
- username - Имя пользователя; 
- email - Электронная почта;
- role - права;
- first_name - Имя;
- last_name - Фамилия;
- avatar - Аватар;
- phone - Номер телефона;
- telegram - Аккаунт телеграм;
- max_messenger - Аккаунт MAX
- is_active - Статус аккаунта;

Разрешения: 
- Реализованы кастомные разрешения для пользователя с ролью MPDERATOR
- Смотреть списки пород, собак и отзывов могут все пользователи
- Смотреть подробную информацию о породе, собаке и отзыве могут все пользователи
- Создавать, изменять, удалять карточку могут только авторизованные пользователи
- Список пользователей могут видеть все авторизованные пользователи
- Регистрироваться на ресурсе может любой пользователь
- Подробную информацию о пользователе могут видеть только авторизованные пользователи
- Изменять информацию о пользователе может только сам пользователь
- Удалять пользователя может только администратор
